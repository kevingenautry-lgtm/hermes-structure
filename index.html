Architecture

How retrieval works

🔀 Fusion Search Pipeline

User Query → 4-Signal Retrieval → RRF Fusion (k=60) → Cross-Encoder Rerank → Recency Weighting → Synthesis + Gap Analysis
🧠 Vector — ChromaDB / BGE-small 🔤 Keyword — SQLite FTS5 / BM25 🏷️ Entity — KG SQLite lookup 🔗 Graph — KG paths + connections

🔄 Auto-Reasoning

Chat Sessions → Cron 12h → LLM Extract → Contradiction Check → concepts/ page

Extracts: preferences, decisions, facts, deductions, trends. Flags contradictions with past observations.

📡 Signal Detector

In-session message → detect_signals() → signals/{profile}.md → KG typed edge

Real-time capture. No cron wait. One LLM call per detection. Typed edges created immediately.

Usage

Code samples

Fusion Search
# Multi-signal retrieval from _fusion import fusion_search results = fusion_search("scalping EURUSD", top_k=5) # Filter by profile results = fusion_search("REITs", profile="nami") # Chat display answer = fusion_search("apa pasal EURUSD", show=True)
Save to Wiki
from _save import save_page result = save_page( title="Scalping Strategy", body="Robin scalps EURUSD 9-11am", profile="robin", relations=[("Robin", "trades", "EURUSD")] ) # Auto: detect type, extract # entities, update FTS5+KG+log
Synthesis + Gaps
from _synthesize import synthesize answer = synthesize("Robin strategy?") # → 📌 Summary # → ✅ Knows with 📍Source[N] # → ⚠️ Gaps + confidence raw = synthesize("Robin strategy?", raw=True) # raw["answer"], raw["gaps"]
Signal Detector
from _wikisignal import capture_signal # Quick capture, no LLM capture_signal("preference", "Prefers async Python", "zorro", ["coding"]) # → signals/zorro-{date}.md # → KG edge: zorro → prefers
Typed Edges
from _kg import summarize_profile, add_typed_edge # Profile summary print(summarize_profile("robin")) # Trades: EURUSD | Prefers: Scalping # Add typed edge add_typed_edge("robin", "EURUSD", "trades")
